## 1. プロダクト概要

### 1.1 プロダクト名
Bookmarkly

### 1.2 コンセプト
「溜めっぱなし」を「ちゃんと使える」に変えるブックマーク管理アプリ。

URLとテキストメモを一元管理し、AIによる自動タグ付けで整理コストをゼロに近づける個人向けWebアプリ。

### 1.3 解決する課題

| 課題 | 詳細 |
|------|------|
| URLが溜まって見返さない | 保存したきり放置。未読管理ができていない |
| タグ・分類が面倒 | 手動でカテゴリ分けする気になれず、無秩序に溜まる |
| 後で読もうとして忘れる | 未読リストが見えない、リマインドもない |
| Slackに流れてしまう | 個人チャンネルに投げたURLが埋もれて消える |

## 2. ターゲットユーザー

- 個人利用（主に自分自身）
- 技術記事・ニュース・書籍など多様なコンテンツを日常的に保存する人
- スマホからサッと登録、PCでまとめて整理、という使い方をする人

## 3. 機能要件

### 3.1 Phase 1（MVP）

#### 3.1.1 ブックマーク登録
- URLあり＋メモ（コメント）での登録
- URLなし＋メモのみ（純粋なテキストメモ）での登録
- URL入力時にOGP（タイトル・サムネイル）を自動取得
- タイトルは手動編集可能

#### 3.1.2 未読・既読管理
- ブックマークは登録時に「未読」状態
- 既読にするワンタップ操作
- ステータス: `unread` | `read` | `archived`

#### 3.1.3 ダッシュボード
- 未読リストをページトップに目立つ形で表示
- 最近追加したブックマーク一覧を表示

#### 3.1.4 全ブックマーク一覧
- 登録済みブックマークの一覧表示
- タイトル・メモ・タグによる全文検索

### 3.2 Phase 2

#### 3.2.1 ユーザー認証
- メールアドレス＋パスワードによる独自実装
- ログイン後はJWTで認証状態を管理
- Phase 1のブックマークデータは `user_id = NULL` で保持し、Phase 2でFK制約を付与

#### 3.2.2 AIによる自動タグ付け
- 登録時にURL or メモの内容をもとにClaude APIがタグを自動生成
- タグは手動で編集・追加も可能

#### 3.2.3 フィルタリング・絞り込み
- タグ別・未読のみ・コンテンツ種別でフィルタ
- コンテンツ種別の自動判定

### 3.3 Phase 3

- アーカイブ機能（古い既読をアーカイブに移動）
- ランダム再表示（「今日の積読」的機能）

## 4. 非機能要件

### 4.1 UX方針
- **スマホファースト**: スマホから素早く登録できることを最優先
- **入力コスト最小化**: URLを貼るだけでタイトル・タグが自動補完
- **PCでの整理体験**: PC表示では一覧性を高め、タグ編集・フィルタがしやすいレイアウト

### 4.2 パフォーマンス
- OGP取得はサーバーサイドで非同期処理し、登録レスポンスをブロックしない
- AIタグ付けは登録後に非同期で実行（登録完了後にタグを後付け更新）

### 4.3 セキュリティ
- パスワードはハッシュ化して保存（bcrypt相当）
- JWTのシークレットキーは環境変数で管理

## 5. データモデル

### 5.1 Bookmark

| カラム | 型 | 備考 |
|--------|----|------|
| id | TEXT (UUID) | PK |
| user_id | TEXT | NULL許容（Phase 1）、Phase 2でFK制約 |
| url | TEXT | NULL許容（メモのみの場合） |
| title | TEXT | OGP自動取得 or 手動入力 |
| memo | TEXT | ユーザーのコメント |
| tags | TEXT | JSON配列として保存 |
| status | TEXT | `unread` / `read` / `archived` |
| content_type | TEXT | `article` / `news` / `book` / `product` / `memo` / `other` |
| thumbnail_url | TEXT | OGPから取得 |
| created_at | TEXT (ISO8601) | |
| updated_at | TEXT (ISO8601) | |

## 6. 画面構成

| パス | 画面名 | 概要 |
|------|--------|------|
| `/` | ダッシュボード | 未読リスト・最近追加したブックマーク |
| `/bookmarks` | 一覧 | 全ブックマーク・検索・フィルタ |
| `/bookmarks/entry` | 新規登録 | URL入力→OGP取得・メモ入力・タグ確認 |
| `/bookmarks/:id` | 詳細・編集 | ブックマーク詳細表示と編集 |
| `/login` | ログイン | メールアドレス＋パスワードでログイン。パスワード表示トグルあり。未登録者向けの新規登録リンクを表示 |
| `/register` | アカウント登録 | メールアドレス＋パスワード＋パスワード確認で登録。パスワード不一致バリデーションあり。既存ユーザー向けのログインリンクを表示 |

## 8. 開発方針

- 個人開発のため、仕様・テストは最低限のシンプルな範囲に留める
- VS Code の Dev Container で開発環境を統一する
- 画面UIは本実装前にHTMLモックを作成してから実装に入る
