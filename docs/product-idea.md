# ブックマークアプリ 構想メモ

## コンセプト

> 「溜めっぱなし」を「ちゃんと使える」に変えるブックマーク管理アプリ

Slackの個人チャンネルや適当なメモにURLを貼り付けて流れてしまう、という日常の痛みを解消する。  
URLだけでなく、テキストメモも一緒に管理でき、AIが自動でタグ付けしてくれる個人向けWebアプリ。

---

## 解決する課題

| 課題 | 詳細 |
|------|------|
| URLが溜まって見返さない | 保存したきり放置。未読管理ができていない |
| タグ・分類が面倒 | 手動でカテゴリ分けする気になれず、無秩序に溜まる |
| 後で読もうとして忘れる | 未読リストが見えない、リマインドもない |
| Slackに流れてしまう | 個人チャンネルに投げたURLが埋もれて消える |

---

## ターゲットユーザー

- **自分自身（個人用）**
- 技術記事・ニュース・Amazonの書籍など多様なコンテンツを日常的に保存する人
- スマホからサッと登録、PCでまとめて整理、という使い方をする人

---

## 主な機能

### コア機能（MVP）

- **ブックマーク登録**
  - URLあり＋メモ（コメント）
  - URLなし＋メモのみ（純粋なテキストメモとして）
  - URLを入力するとOGP（タイトル・サムネイル）を自動取得
- **未読・既読管理**
  - 未読リストをダッシュボードのトップに表示
  - 既読にするワンタップアクション
- **AIによる自動タグ付け**
  - 登録時にURLまたはメモの内容をもとにAIがタグを自動生成
  - タグは手動で編集・追加も可能
- **検索**
  - タイトル・メモ・タグで全文検索

### 追加機能（v2候補）

- フィルタリング（タグ別・未読のみ・コンテンツ種別）
- アーカイブ機能（古い既読をアーカイブに移動）
- ランダム再表示（「今日の1件」的な機能）

---

## コンテンツ種別

アプリが想定するブックマークの種類：

- 技術記事・ドキュメント
- ニュース・時事
- Amazon書籍（「後で買うかも」メモ）
- 商品・ショッピング
- テキストメモのみ（URLなし）

---

## UX方針

- **スマホファースト**：スマホから素早く登録できることを最優先
- **入力コストを最小化**：URLを貼るだけでタイトル・タグが自動補完される
- **PCでの整理体験**：PC表示では一覧性を高め、タグ編集・フィルタリングがしやすいレイアウト
- **ログイン不要でも試せる**（初期はシンプルに。認証は後回しでもOK）

## 技術スタック

| レイヤー | 技術 |
|----------|------|
| フロントエンド | Vue 3 |
| バックエンド（API） | Hono |
| データベース | Turso（SQLite互換・エッジDB） |
| AI タグ付け | Claude API（Anthropic） |
| セッション管理 | JWT |
| OGP取得 | サーバーサイドでスクレイピング |

## データモデル（草案）

```
Bookmark
  - id
  - user_id (nullable ※Phase1はNULL許容、Phase2でNOT NULL + FK制約を付与)
  - url (nullable)
  - title (OGPから自動取得 or 手動)
  - memo (ユーザーのコメント)
  - tags (AI生成 + 手動)
  - status: unread | read | archived
  - content_type: article | news | book | product | memo | other
  - thumbnail_url (OGPから)
  - created_at
  - updated_at
```

---

## 画面構成（草案）

```
/ (ダッシュボード)
  - 未読リスト（上部に目立つ形で表示）
  - 最近追加したもの

/bookmarks
  - 全ブックマーク一覧
  - タグ・ステータスでフィルタ
  - 検索バー

/bookmarks/new
  - URL入力 → OGP自動取得
  - メモ入力
  - AIタグ確認・編集

/bookmarks/[id]
  - 詳細表示・編集
```

## 開発フェーズ

### Phase 1（MVP）
- ブックマーク登録（URL+メモ / メモのみ）
- OGP自動取得
- 未読・既読管理
- ダッシュボード（未読リスト表示）
- 基本的な検索

### Phase 2
- ユーザー登録・認証機能（メール＋パスワードハッシュ独自実装）
- AIによる自動タグ付け
- タグフィルタ・絞り込み
- コンテンツ種別の自動判定

### Phase 3
- アーカイブ機能
- ランダム再表示（「今日の積読」的機能）
- 認証機能（必要になったら）

---

## 認証方式

- **独自実装**（外部サービス不使用）
- `users` テーブルにメールアドレス＋パスワードハッシュを保存
- ログイン後はセッション or JWTで認証状態を管理

```
User
  - id
  - email
  - password_hash
  - created_at
  - updated_at
```
